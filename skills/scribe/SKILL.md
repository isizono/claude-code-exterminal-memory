# scribe

cc-memoryの記録（decisions, logs, materials）を元に、外部共有可能なドキュメントを生成する。

## 1. 対象アクティビティの選択

1. 引数で `activity_id` が指定されていればそのまま使う
2. 指定されていなければ `get_activities(status="completed")` で候補を表示し、ユーザーに選んでもらう
3. 複数のアクティビティをまとめて1つのドキュメントにすることもできる。ユーザーに確認する

## 2. [執筆]アクティビティの作成

対象が決まったら、ドキュメント生成作業を追跡するアクティビティを作成し、`check_in` する。

- タイトル: `[執筆] {仮タイトル}`（構成確定後に `update_activity` で更新する）
- タグ: `intent:document` + ソースアクティビティの `domain:` タグを引き継ぐ
- description: ソースアクティビティのIDと目的を記載

## 3. 素材の収集

ソースアクティビティごとに以下を収集する:

- `check_in(activity_id=...)` でアクティビティ情報を取得
- check-in結果の `related_topics` から `topic_id` を取得する
- `get_decisions(topic_id=...)` で決定事項を取得
- `get_logs(topic_id=...)` で議論経緯を取得
- `search(type_filter="material")` / `get_material` で資材を取得

必要に応じて、ソースアクティビティのタグで `search` し、関連する別トピックの情報も補足的に収集する。

## 4. ボリューム判断と構成

収集した素材の量を確認する。

**通常の場合**: そのままドキュメント1本として書く。構成案を提示してユーザーの確認を取る。確定したらアクティビティのタイトル・descriptionを更新する。

**ボリュームが大きい場合**: 分割が必要と判断したら、分割案を提示してユーザーとすり合わせる。合意後、各ドキュメントに対して追加の[執筆]アクティビティを `add_activity` で作成する。

## 5. 執筆

`intent:document` のtag notesに従ってドキュメントを生成する。

tag notesが未設定の場合のデフォルト構成:

```markdown
---
date: {日付}
source_activities: [{activity_id一覧}]
---

# {タイトル}

## 背景

{なぜこの決定が必要だったのか。動機・制約・前提条件}

## 決定事項

{何を決めたか。具体的な内容}

## 代替案と却下理由

{検討したが採用しなかった選択肢と、その理由}
```

重要なルール:
- cc-memoryにアクセスできない読者を前提とする。内部ID（decision #123等）を参照せず、内容を直接記述する
- ソースが不足している箇所には `<!-- TODO: 要確認 -->` マーカーを付けて書き進める
- ドキュメント間参照はファイル名で行う

## 6. 出力と確認

1. 生成したドキュメントをユーザーに提示する
2. ユーザーのフィードバックを受けて修正する
3. 最終確認が取れたらファイルに書き出す
4. [執筆]アクティビティをcompletedにする
5. 分割ありの場合: 残りの[執筆]アクティビティの存在をユーザーに伝える

## 注意事項

- ユーザーの確認なしに書き始めない。構成案の合意を必ず取る
- 「スキルは骨組み、tag notesは味付け」の原則に従い、執筆時の振る舞いの詳細はintent:documentのtag notesに委ねる
- エクスポート先のファイルパスは必ずユーザーに確認する
