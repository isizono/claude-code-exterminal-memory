# ポストモーテム

completedアクティビティの行動を時系列で振り返り、ユーザーと対話しながら教訓を抽出・永続化する。

## 1. 対象アクティビティの選択

1. 引数で `activity_id` が指定されていればそのまま使う
2. 指定されていなければ `get_activities(status="completed", limit=15)` で一覧を表示し、ユーザーに選んでもらう

## 2. ポストモーテムアクティビティの作成

対象が決まったら:

1. 新しいアクティビティを作成:
   - タイトル: `[振り返り] {元アクティビティ名}のポストモーテム`
   - タグ: `intent:discuss`, `振り返り`, + 元アクティビティの `domain:` タグを引き継ぐ
   - `related`: 元アクティビティとリレーションを張る（`[{"type": "activity", "ids": [元アクティビティID]}]`）
2. check-in する

## 3. データ収集とステップ分解

対象アクティビティに関連するデータを収集する:

- check-in結果の `related_topics` から `topic_id` を取得する。`related_topics` が空の場合は、アクティビティのタグで `search` するか、ユーザーに関連トピックを確認する
- `get_logs(topic_id=...)` でログを取得
- `get_decisions(topic_id=...)` で決定事項を取得
- `get_map(entity_type="activity", entity_id=...)` で関連エンティティを探索し、追加の情報源がないか確認する
- `search(type_filter="material")` / `get_material` で資材を確認

### データ不足の判定

ログが2件以下、かつdecisionも1件以下の場合:

```
データが少なくて、ステップ分解が難しいかも。
ログ: {N}件 / decision: {N}件

このまま進める？（ユーザーが口頭で補足しながらやることもできる）
```

ユーザーが続行を選んだ場合は、ユーザーに「何をやったか」を聞きながら手動でステップを組み立てる。

### ステップ一覧の作成

ログを時系列で読み、行動の切れ目でステップに分解する。以下のフォーマットで**全ステップの一覧を先に提示**する:

```
## ステップ一覧

1. {ステップ名} — {1行の概要}
2. {ステップ名} — {1行の概要}
3. ...

この分け方で進める？追加・統合・分割があれば言って。
```

ユーザーの確認を待ってから次に進む。

## 4. ステップごとのレビュー

エージェントは**材料を並べる係**に徹し、評価はユーザーに先に出してもらう。1ステップずつ、以下の構成で提示する:

```
### ステップ {N}: {ステップ名}

**何があったか**:
{ログから読み取れる事実を時系列で要約}

**補足コンテキスト**:
{検討した代替案、関連decision、当時の制約条件など。なければ省略}

ここ振り返ってみてどう？
```

- エージェントは事実と文脈を提示するが、**よかった/悪かったの評価は書かない**
- ユーザーが意見を述べ始めたら、エージェントも自分の見解を出してよい
- 「ここは別のやり方もあった？」「この判断の背景って何だった？」など、振り返りを深める問いかけを使う

ユーザーの発言は次ステップに進む前にログとして記録する。

## 5. 反省ポイントまとめ

全ステップのレビューが終わったら、反省ポイントを一覧で出す:

```
## 反省ポイントまとめ

### よかったこと
- ...

### 改善したいこと
- ...（各項目に「なぜ起きたか」の1行分析を添える）
```

この一覧を `add_material` でポストモーテムアクティビティに保存する。

## 6. 教訓の永続化

最後に、改善したいことの中から**次に活かせる教訓**をどう永続化するか議論する。

永続化の手段はケースバイケース。以下は選択肢の例:
- **tag notes**: 特定のintentやdomainに遭遇したときに自動注入されるルール
- **decision**: トピックに紐づく合意事項
- **auto memory (feedback)**: Claude Codeのセッション横断で適用される行動ルール
- **CLAUDE.md**: プロジェクト全体に適用されるルール
- **新しいintentタグ**: 行動パターンが根本的に異なる場合

エージェントは選択肢を提示するが、**何をどこに落とすかはユーザーが決める**。

永続化が完了したら、ポストモーテムアクティビティの description に結論を追記し、ステータスを completed にする。

## 注意事項

- 急がない。1ステップずつ丁寧に。ユーザーが「次」と言うまで進まない
- エージェントの評価を押し付けない。「ここは問題だった」ではなく「ここ気になったんだけど、どう思う？」
- ユーザーの発言はできるだけ原文に近い形でログに残す
- セッション跨ぎの可能性があるので、進捗はこまめにログ記録する
