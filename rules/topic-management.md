# トピック管理ルール

## 原則: 1トピック = 1つの論点・問題・機能

トピックは「この会話、一言で何の話？」に答えられる粒度にする。

## 迷ったら切る

トピックを切るべきか迷ったら、**切る方を選ぶ**。

- 細かすぎて困ることはない（後でまとめられる）
- 粗いと分割が大変
- 「まだ同じトピックかな...」と思った時点で、たぶん別トピック

## 新しいトピックを切るタイミング

- 新しい問題・バグが発覚した
- 別の機能・コンポーネントの話になった
- ユーザーが明示的に話題を変えた（「次に〜」「別件だけど〜」）
- 元のトピックのdescriptionに含まれない内容になった

## 子トピックの活用

大きなテーマの下に具体的な論点がある場合は親子関係を使う。

例:
```
親: Stopフック実装
  ├─ 子: 無限ループの原因調査
  ├─ 子: メタタグのパース処理
  └─ 子: first_topicからの移動処理
```

## トピック作成時の注意

- **具体的な名前をつける**: 「〇〇の設計」のような曖昧な名前は避ける
- **descriptionを書く**: 後で「このトピック何だっけ？」とならないように
- **parent_topic_idを活用**: 関連するトピックは親子関係でつなげる
