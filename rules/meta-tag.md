# メタタグ出力ルール

## 応答の最後にメタタグを出力

毎ターンの応答の最後に、以下の形式でメタタグを出力すること。

### フォーマット

```html
<!-- [meta] project: {プロジェクト名} (id: {プロジェクトID}) | topic: {トピックタイトル} (id: {トピックID}) -->
```

## ルール

1. **必ず出力する**: 応答の最後にメタタグを出力しないとStopフックでブロックされる
2. **HTMLコメント形式**: ユーザーには見えないが、hookでパース可能
3. **トピック変更時**: 新しいtopic_idを出力する
4. **新規トピック時**: add_topicで作成してからそのIDを出力する

## 例

### 基本例

```html
<!-- [meta] project: claude-code-exterminal-memory (id: 2) | topic: [実装]Stopフック並列実行時のtopic_id競合バグ (id: 93) -->
```

### 様々なプロジェクト・トピックの例

```html
<!-- [meta] project: claude-code-exterminal-memory (id: 2) | topic: [議論]プラグインキャッシュとコンポーネント読み込み問題 (id: 95) -->

<!-- [meta] project: LLM技術ブログ執筆 (id: 6) | topic: [議論]第4章: コンテキストエンジニアリングの必要性 (id: 99) -->

<!-- [meta] project: drift (id: 5) | topic: [設計]ローカルテスト戦略の検討 (id: 77) -->
```

## 注意

- プロジェクトIDとトピックIDは必ず正確な数値を使う
- トピックが変わる場合は、前のトピックに決定事項があるか確認してから変更する
